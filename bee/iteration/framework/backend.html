后端项目

1. 开发环境

  • JDK 17（源码级别 8、编译级别 8）

  • 构建工具： Gradle 7.4.2

  • 项目框架：Spring Boot 2.7.0

Gradle 版本和 Spring Gradle 插件版本存在不兼容的情况，需要注意。 Gradle 7.4.2 和 spring-boot-gradle-plugin 2.7.0 可正常匹配

2. 应用

2.1. 主应用

依赖如下：

build.gradle
    // web 服务器
    implementation "org.springframework.boot:spring-boot-starter-web"
    implementation "org.springframework.boot:spring-boot-starter-actuator"
    // SBA 客户端
    implementation "de.codecentric:spring-boot-admin-starter-client"
    // 在 SBA 中基于 JMX 修改 MBean
    implementation "org.jolokia:jolokia-core"
    // OpenAPI 3 文档支持：https://github.com/springdoc/springdoc-openapi
    // http://bee.peacetrue.local/api/swagger-ui/index.html /api/v3/api-docs
    implementation "org.springdoc:springdoc-openapi-ui:1.6.11"
    // 从 javadoc 读取接口描述：https://springdoc.org/#javadoc-support
    implementation "org.springdoc:springdoc-openapi-javadoc:1.6.11"
    annotationProcessor "com.github.therapi:therapi-runtime-javadoc-scribe:0.15.0"

    runtimeOnly "io.micrometer:micrometer-registry-prometheus"
    implementation "org.springframework.boot:spring-boot-starter-aop"

    // Bean 验证
    implementation "org.springframework.boot:spring-boot-starter-validation"
    // 响应结果支持 https://peacetrue.github.io/peacetrue-result/index.html
//    implementation "com.github.peacetrue.result:peacetrue-result-exception-support"

2.2. 监控应用

依赖如下：

build.gradle
    implementation "org.springframework.boot:spring-boot-starter-web"
    implementation "de.codecentric:spring-boot-admin-starter-server" // SBA 服务端

3. 构建

使用 application 插件打包后， 部署到远程服务。

以 main 为例说明， monitor 与此类似。

3.1. 部署包结构

运行 ./gradlew installBootDist， 在 build/install 目录下生成部署包， 文件结构如下：

main-boot
├── bin
│   ├── main
│   └── main.bat
└── lib
    └── main-1.0.0.jar

3.2. 运行脚本

运行脚本名称默认与项目名相同。

调用脚本，脚本会自动根据上面的目录结构，找到执行的 jar 包：

$ cd main-boot/bin
$ ./main

脚本后面的参数作为程序参数，会传递给 main 方法。 如需更改 JVM 参数，可使用 JAVA_OPTS 环境变量：

$ export JAVA_OPTS='"-Xmx1G" "-Xms1G"'
$ ./main

关于 $JAVA_OPTS 的调用方式：

$ export JAVA_OPTS='"-Xmx1G" "-Xms1G"'
$ java $JAVA_OPTS -jar backend-1.0.0-SNAPSHOT.jar
错误: 找不到或无法加载主类 "-Xmx1G" "-Xms1G"
原因: java.lang.ClassNotFoundException: "-Xmx1G" "-Xms1G"

# 正确的调用方式如下：
$ eval "java $JAVA_OPTS -jar backend-1.0.0.jar"
