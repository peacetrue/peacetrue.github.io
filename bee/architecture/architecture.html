架构

前后端分离。

1. 项目划分

  • 后端主应用：提供功能接口

  • 后端监控应用：监控后端主应用运行状态及异常告警

  • 前端应用：提供功能界面展示

2. 部署模式

2.1. 本地

Figure 1. 本地部署模式
应用端口

  • 前端应用：3000

  • 后端监控应用：8080

  • 后端主应用：8081

本地使用地址： http://bee.peacetrue.local 。

添加 hosts 条目：

/etc/hosts
echo '127.0.0.1 bee.peacetrue.local' >> /etc/hosts

Nginx 配置：

bee.peacetrue.local
server {
    listen       80;
    server_name  bee.peacetrue.local;

    root html/bee;
    index index.html;

    location / {
        proxy_pass http://localhost:3000/;
    }

    location /monitor/ {
       proxy_pass http://localhost:8080/;
    }

    location /api/ {
       proxy_pass http://localhost:8081/;
    }

}

2.2. 远程

远程指服务端。

Figure 2. 远程部署模式
应用端口

  • 后端监控应用：8080

  • 后端主应用：8081

远程使用地址： http://bee.peacetrue.cn 。

Nginx 配置：

bee.peacetrue.cn
server {
    listen       80;
    server_name  bee.peacetrue.cn;

    root html/bee;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /monitor/ {
       proxy_pass http://localhost:8080/;
    }

    location /api/ {
       proxy_pass http://localhost:8081/;
    }
}

4. 接口协议

后端接口基于 HTTP 协议， 遵循 REST API 规范。

4.1. 接口版本规范

使用响应 媒体类型 定义接口版本， 格式为 application/vnd.bee.version+json。

version 表示版本号， 例如：application/vnd.bee.v1+json、application/vnd.bee.v2+json 。

为什么不使用 URI 形式的版本？以及如何简单地在服务端和客户端升级版本？

  • https://www.baeldung.com/rest-versioning

  • https://www.baeldung.com/spring-rest-custom-media-type

4.2. 响应结果

前端根据后端响应状态处理返回数据。 非 200 状态码，直接提示 statusText 值。
