BeanMap

BeanMap 是 LinkedHashMap<String, Object> 的子类， 表示一个泛化的 Java Bean 对象。

如果说一个方法接收 BeanMap 作为参数， 实际上表示它可以接收 Map<String, Object>， BeanMap 的好处在于可以避免泛型转换。

1. 格式

BeanMap 有 2 种格式：层级和扁平。

假设，你有如下 User 类：

    @Data
    @Builder(toBuilder = true)
    @NoArgsConstructor
    @AllArgsConstructor
    private static class User {
        private Long id;
        private String name;
        private String password;
        private List<Role> roles;
        private Employee employee;
    }

    @Data
    @Builder(toBuilder = true)
    @NoArgsConstructor
    @AllArgsConstructor
    private static class Role {
        private Long id;
        private String name;
        private List<String> tags;
    }

    @Data
    @Builder(toBuilder = true)
    @NoArgsConstructor
    @AllArgsConstructor
    private static class Employee {
        private Long id;
        private String name;
        private List<String> tags;
    }

层级格式示例如下：

tiered
{
  "id" : null,
  "name" : "admin",
  "password" : "",
  "roles" : [ {
    "id" : 1,
    "name" : "admin",
    "tags" : [ "read", "write" ]
  }, null, {
    "id" : null,
    "name" : "user",
    "tags" : null
  }, {
    "id" : 3,
    "name" : null,
    "tags" : null
  } ],
  "employee" : {
    "id" : 1,
    "name" : "Jone",
    "tags" : [ "good", "better" ]
  }
}

扁平格式示例如下：

flat
{
  "id" : null,
  "name" : "admin",
  "password" : "",
  "roles[0].id" : 1,
  "roles[0].name" : "admin",
  "roles[0].tags[0]" : "read",
  "roles[0].tags[1]" : "write",
  "roles[1]" : null,
  "roles[2].id" : null,
  "roles[2].name" : "user",
  "roles[2].tags" : null,
  "roles[3].id" : 3,
  "roles[3].name" : null,
  "roles[3].tags" : null,
  "employee.id" : 1,
  "employee.name" : "Jone",
  "employee.tags[0]" : "good",
  "employee.tags[1]" : "better"
}

相信读者已经区分出，这 2 种格式的区别了。

2. 属性遍历

假设，你有一个上述层级格式的数据，可以通过如下方式遍历所有属性：

        BeanMapUtils.walkTree(tiered, (path, value) -> log.info("path: {}, value: {}", path, value));

输出日志如下：

path: id, value: null
path: name, value: admin
path: password, value:
path: roles[0].id, value: 1
path: roles[0].name, value: admin
path: roles[0].tags[0], value: read
path: roles[0].tags[1], value: write
path: roles[1], value: null
path: roles[2].id, value: null
path: roles[2].name, value: user
path: roles[2].tags, value: null
path: roles[3].id, value: 3
path: roles[3].name, value: null
path: roles[3].tags, value: null
path: employee.id, value: 1
path: employee.name, value: Jone
path: employee.tags[0], value: good
path: employee.tags[1], value: better
