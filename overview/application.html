应用组装

本文介绍如何基于应用拆分与组装实现代码复用。

1. 应用构件

一个能够运行的 Java Web 应用，实现代码通常包含以下三部分内容：

  1. Servlet 容器

  2. 框架层代码

  3. 业务层代码

Servlet 容器和框架层代码称为应用外壳，应用外壳本身可以独立运行，但不包含业务功能； 业务层代码称为应用内核，用于实现具体业务功能。 应用外壳和应用内核配合运行，成为一个有效的应用；只运行应用外壳则是一个空应用。

当然，不必局限于 Java Web 应用，从更高的层次来看，应该是如下结构：

已实现的应用外壳如下

  1. peacetrue-application

已实现的应用内核如下

  1. peacetrue-dictionary

2. 应用组装

在一些资源库中实现应用外壳程序，分版本区别其中的差异； 在另一些资源库中实现应用内核程序，分版本区别其中的差异； 最终选择合适版本的应用外壳和应用内核组成新应用：

实际的组装过程具体如何操作呢？ 直接在应用外壳中引用应用内核，可以临时使用，如何在新项目中实现组装呢？ 拷贝应用外壳，修改应用名及相关信息，也是一种方式，但会有些繁琐， 以下介绍基于模板的方式。

3. 归纳演绎

将已有应用外壳归纳为 目录模板， 再通过 目录模板引擎 演绎出（生成）新应用。 当然，除了应用外壳，其他但凡觉得繁琐的应用构件，都可以归纳成目录模板。

4. 实现过程

  1. 实现应用外壳

  2. 实现应用内核

  3. 应用内核集成应用外壳测试（拷贝或者连接）

  4. 发布应用内核

  5. 应用外壳集成已发布应用内核测试

  6. 应用外壳还原成未集成状态

  7. 制作应用外壳目录模板

  8. 目录模板引擎 + 外壳目录模板 + 已发布应用内核 + 模板变量 → 新项目
