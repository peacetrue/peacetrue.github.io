目录模板

当开发某个功能时，意识到该功能和之前做过的某些功能， 在代码上具有相似性，此时可以考虑，基于之前的代码，制作目录模板。

模板引擎不同，编写模板的语法也不同， 目前组件内置的模板引擎基于 Velocity 实现， 文中涉及的示例，也均使用 Velocity 语法实现。

模板语法虽然有所差异，但最终表现效果却是类似的，都包括：

  • 获取/设置变量值

  • 判断

  • 循环

  • 调用方法

  • 声明/调用宏

目录模板的具体内容是不受限制的，可以实现各种各样的功能，满足了业务的复杂性。 最终制作完成后，会得到一个目录，但目录不便于在网络上传输，可以打成压缩包或 jar 包。 以 Antora 目录模板 为例，打包命令如下：

Antora 目录模板
#tar -cvf antora.tar antora
zip -rq antora.zip antora
jar -cvf antora.jar antora

为了方便使用，同时应该说明目录模板中涉及变量的含义。

如果是 Java 开发者，还可以扩展 Velocity 的功能， 为编写目录模板提供更多好用的工具。
