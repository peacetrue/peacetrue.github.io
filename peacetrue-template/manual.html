手册

1. 基本工作流程

目录模板引擎渲染时序图
@startuml
autonumber
客户端 -> 目录模板引擎: 渲染目录模板
目录模板引擎 -> 资源集合加载器: 获取目录模板下资源集合
目录模板引擎 <- 资源集合加载器: 返回资源集合
loop 循环处理资源
    alt 资源为模板
        目录模板引擎 -> 模板引擎: 渲染模板
        模板引擎 -> 模板引擎: 生成结果
        目录模板引擎 <- 模板引擎: 返回
    else 资源不为模板
        目录模板引擎 -> 目录模板引擎: 拷贝资源到结果
    end
end
客户端 <- 目录模板引擎: 返回
@enduml

3. 模板引擎 TemplateEngine

模板引擎是一个接口，用于避免直接依赖具体模板引擎。内置了 1 个实现：

  • VelocityTemplateEngine：基于 Velocity 实现的模板引擎

开发者也可自定义其他实现，例如：基于 FreeMark 实现。

3.1. Velocity 模板引擎

如果基于 VelocityTemplateEngine 渲染模板，构建目录模板时需要遵循 Velocity 模板的语法。 VelocityTemplateEngine 默认启用了 Velocity 工具 的支持。 同时，笔者还扩展了 Velocity 工具，具体功能可参考 待完善。

4. 资源集合加载器 ResourcesLoader

资源集合加载器用于获取目录模板下所有资源。 区别于 Spring 的 ResourceLoader，它只能获取单个资源。

内置了 2 个实现：

  • FileSystemResourcesLoader：文件系统资源加载器

  • ClasspathResourcesLoader：类路径资源加载器

5. 目录模板引擎 DirectoryTemplateEngine

目录模板引擎渲染时，接收 4 个参数：

templateLocation

目录模板位置。 文件路径使用 file: 前缀，例如：file:/path/to/template， 类路径使用 classpath: 前缀，例如：classpath:/path/to/template。 当目录模板为文件时，位置可以指定为目录、jar 包（路径以 .jar 结尾）或 zip 包（路径以 .zip 结尾）。 不支持其他压缩包格式，因为 Java 原生只提供了 zip 包的解析，如果是其他压缩包格式，请自行转换为 zip 包。 Java 开发者推荐使用 jar 包，非 Java 开发者推荐使用 zip 包。
