== 数据结构格式
数据结构使用以下表格进行定义：

|===
|名称 |描述 |类型 |可空 |备注

|_属性名称_
|_属性描述_
|_{数据类型}_
|_是/否_
|_需要注意的事项_
|===


== 通用数据结构

[[标准响应数据]]
=== 标准响应数据<__待定类型__>
所有接口都返回此结构的数据

|===
|名称 |描述 |类型 |可空 |备注

|code
|编码
|整数/字符串
|否
|编码用于指示响应状态（是正常还是异常）

|message
|描述
|字符串
|否
|对应响应状态的具体描述信息

|[[负载数据]]data
|负载数据
|_待定类型_
|是
|具体类型根据实际情况决定
|===

=== 签名后参数
|===
|名称 |描述 |类型 |可空 |备注

|data
|原始参数
|字符串
|否
|

|appId
|应用标识
|字符串
|否
|

|sign
|签名值
|字符串
|否
|
|===

=== 参数
|===
|名称 |描述 |类型 |可空 |备注

|name
|参数名
|字符串
|否
|

|value
|参数值
|基本类型
|否
|

|type
|参数类型
|任意类型
|否
|具体类型根据实际情况而定
|===

|否
|

|content
|数据
|_待定类型_ 集合
|否
|
|===
