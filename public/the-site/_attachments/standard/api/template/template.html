1. 认证方式

客户端调用接口时，需要提供两个额外参数：应用标识和签名， 服务端会校验签名参数是否正确， 正确视为认证通过，否则视为认证失败。

签名过程中会涉及到应用标识和应用秘钥， 应用标识用于区分不同的客户端， 应用秘钥用于认证身份。

1.3. 接口参数

在声明接口参数时，只需要声明原始参数， 不需要声明签名后参数，签名后参数的结构都是相同的：

{
"data":"{原始参数}",
"appId":"{应用标识}",
"sign":"{签名值}"
}

签名后参数仅用于认证，认证通过后只需要使用原始参数。

2. 数据类型

在描述接口时需要使用到数据类型，数据类型包括以下有效值：

2.1. 基础类型

2.1.1. 整数

不带范围

直接使用 整数

带范围

整数(min,max)：min表示最小值、max表示最大值；

  • 不确定最小值：整数(,max)

  • 不确定最大值：整数(min)

2.1.2. 小数

不带范围

直接使用 小数

带范围

小数(M,D)：M 标度，表示小数总位数，D 精度，表示小数部分位数

2.1.3. 字符串

不带范围

直接使用 字符串

带范围

字符串(size)：size表示字符串的最大长度

2.1.4. 布尔值

布尔值作为参数时，默认接收以下有效值：

  • true/false

  • 1/0

  • on/off

2.1.5. 日期

日期作为参数时，默认接收以下有效格式：

  • yyyy-MM-dd

  • yyyy-MM-dd HH:mm:ss

  • 整形毫秒数

日期作为返回数据时，默认输出以下格式：

  • yyyy-MM-dd HH:mm:ss

2.1.6. 集合

串联其他类型一起使用，例如：整数集合、字符串集合等等

2.2. 自定义类型

使用 数据结构格式 声明的类型

2.3. 任意类型

包含以上所有类型

2.4. 类型的表述

  • 可以使用一个具体的类型，例如：整数；

  • 可以使用一个泛化类型，例如：基础类型、任意类型；

  • 可以使用多个类型，例如：整数/字符串。
